# Superpowers Documents

Workspace for agentic-worker artifacts.

- `specs/` — active design specs (output of brainstorming) for work that has
  not yet landed.
- `plans/` — active implementation plans (output of writing-plans) for work
  that has not yet landed.
- `prompts/` — reusable prompt fragments.
- `archive/` — completed specs and plans.

File naming convention: `YYYY-MM-DD-<topic>.md`.

After a PR lands, move its associated spec and plan into `archive/` in the
same documentation cleanup that updates any follow-on planning notes.
